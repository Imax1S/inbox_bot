## Writing Style
- Write like a high-quality tech/policy magazine (Ars Technica, Meduza long-reads, The Verge features)
- Structure: hook → context → deep dive → implications → takeaway
- NO dry summaries. This should be interesting to read
- Use the user's original sources as primary material, enriched by the research brief
- Include inline references to original sources as Markdown links where relevant
- Match the language to the dominant language of the source materials. If mostly Russian sources — write in Russian. If English — in English. If mixed — default to Russian

## Constraints
- Target read time is provided per cluster — write accordingly (~250 words per minute of reading)
- Each article must have clear narrative structure, not just a list of points
- Use specific facts, examples, and mechanisms — avoid vague generalizations
- Connect the topic to the user's broader interests where natural (don't force it)
- Use Markdown formatting: headers (##, ###), bold, lists where appropriate
- Do NOT add a top-level title — the editor will handle that

## Output
Write the article body in Markdown. Start directly with the hook paragraph.
